The PoseCore server is used to be the central hub when you want to control different applications using your body parts as controller (head, hands, feet...).

It can be run in 2 modes, with or without the PoseNet body tracking component

It contains 2 parts

  • A PoseCore communication server over which all body tracking sensors & the clients communicate
  • A PoseNet body tracking component using your local webcamera to fetch your body joint position and send this over the PoseCore communication server

The PoseCore communication server contains both an MQTT srv & a broadcasting support to allow autoconnect of clients
